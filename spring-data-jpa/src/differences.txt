============================================================
DIFFERENCE BETWEEN JPA, HIBERNATE, AND SPRING DATA JPA
============================================================

1. JPA (Java Persistence API)
------------------------------------------------------------
- WHAT: A specification (interface) defined by Oracle.
- PURPOSE: Describes how to map Java objects to database tables.
- NATURE: Just a set of rules/interfaces – provides NO implementation.
- EXAMPLE: javax.persistence.EntityManager, @Entity, @Id.

2. HIBERNATE
------------------------------------------------------------
- WHAT: An ORM (Object-Relational Mapping) framework.
- PURPOSE: Provides the actual implementation of JPA (and adds extra features).
- NATURE: Concrete library that does the actual database operations (SQL generation, caching, etc.).
- EXAMPLE: org.hibernate.Session, Hibernate-specific annotations like @Cascade.

3. SPRING DATA JPA
------------------------------------------------------------
- WHAT: An abstraction layer built on top of JPA.
- PURPOSE: Provides a Repository pattern – you define an interface, Spring generates the implementation at runtime.
- NATURE: Reduces boilerplate code (CRUD methods, pagination, sorting are provided out-of-the-box).
- EXAMPLE: JpaRepository, CrudRepository, derived query methods like findByAuthor().

RELATIONSHIP (How they work together):
------------------------------------------------------------
Spring Data JPA → uses Hibernate (as the JPA provider) → which implements the JPA specification.

ANALOGY:
------------------------------------------------------------
- JPA       = The Recipe (what to do)
- Hibernate = The Chef (does the actual cooking)
- Spring Data JPA = The Kitchen Assistant (makes it easier to order food)

WHY USE SPRING DATA JPA?
------------------------------------------------------------
- No need to write boilerplate CRUD code.
- Automatic implementation of repository interfaces.
- Easy integration with Spring Boot.
- Support for derived queries, pagination, and sorting.

WHEN TO USE HIBERNATE DIRECTLY?
------------------------------------------------------------
- When you need advanced Hibernate-specific features not covered by JPA.
- When you're not using Spring (e.g., plain Java applications).
